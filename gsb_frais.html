Frais

Présentation:

Le projet GSB Frais est un projet de BTS SIO 2 option SLAM réalisées individuellement pour le laboratoire pharmaceutique Galaxy Swiss Bourdin. Le but de ce projet était de créer une application web permettant à des visiteurs médicaux de déclarer leurs frais forfaitaire (déplacement, restauration, nuitée, étapes) et hors forfait via des notes de frais à renseigner depuis l'application web. L'application web devait permettre à un comptable de valider ou non les frais déclarés par les visiteurs médicaux. Le projet a été réalisé en méthode AGILE SCRUM avec notamment l'utilisation d'un backlog et de users stories.

Création:

Ce projet a été l'un des projets principaux de notre année. Nous sommes parties d'une base, l'ancienne application de GSB - frais. Nous avions de plus un cahier des charges à partir duquel nous avons créé le backlog nécessaire au projet. L'application web a été créer en majorité en PHP, HTML et CSS avec notamment le framework Symfony ce qui a aussi impliqué l'utilisation du Twig pour les vues. Le projet repose sur le principe de Modèle Vue Contrôleur (MVC) pour facilité la compréhension du code par un développeur tière.

Choses Appris
Compréhension d'un code Pour effectuer ce projet, nous avions accès au code de l'ancienne application GSB - frais. Avant de commencer à développer la nouvelle application, il fallait déjà comprendre comment fonctionne l'ancienne. Il a donc fallut se plonger dans ce code pour en comprendre ses méthodes et savoir ce qui pouvait être réutilisée.
Analyser un cahier des charges La demande pour le produit a été donnée par un cahier des charges. Il a donc fallut d'abord comprendre chaque besoin, les découper en users stories et les classer par importance. Cette manière de faire se rapproche le plus d'un cas réel, avec un client ayant une demande
L'entraide Sur ce projet, tous les élèves le faisaient en même temps de manière individuelle. De ce fait, tous n'avançaient pas à la même vitesse. L'entraide a donc été important car nous avons tous des capacités différente comme pour le front-end, le back-end, le SQL...
Modélisation de données Ce projet à nécessité la création d'une base de données SQL. Il a donc fallut modéliser cette base de données en fonction des besoins exprimés dans le cahier des charges. Nous avons utilisé l'outil PHPMyAdmin pour la création de cette base de données et avons d'abord créé le schema de la base de données avant de commencer à développer l'application web.
